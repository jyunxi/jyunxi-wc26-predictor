# 2026世界杯智能预测平台

打开 **`index.html`** 即可使用(界面中文,可交互)。六个标签页:

- **Overview** — 夺冠概率、实力榜、每轮晋级概率
- **Match Predictor** — **全部72场小组赛逐场比分预测**:最可能比分、胜/平/负概率条;点击任意比赛展开完整比分概率热力图、预期进球、Top5比分、该场缺阵球员
- **Groups** — 各组积分与出线概率
- **Projected Bracket** — 最可能的淘汰赛路径,32强→决赛逐场预测比分
- **Players** — 48队核心球员:伤病/红黄牌停赛/状态,逐场停赛逻辑,赛事中动态状态值
- **Factors Lab** — 因子实验室:拖动滑块即时重算所有预测

## 模型:Elo + 自适应ML + 三大软因子

1. **Elo实力层** — [World Football Elo Ratings](https://www.eloratings.net) 实时评分;每场新结果按Elo公式更新(K=60,净胜球加成);东道主+100(可调)。
2. **自适应ML校准层** — 贝叶斯逻辑回归,每次更新用本届已赛比赛重新训练,修正热门偏差与进球率;数据越多越偏向数据。
3. **更衣室因子** — 每队的新闻条目(伤病、内讧、士气)按 影响力×可信度 计入,折算为Elo修正(±50,权重可调);还可手动微调每队士气(-5~+5)。
4. **假消息因子** — 可信度<0.7的条目视为"传闻",额外乘以你设定的"传闻信任度"(0~1)。把滑块拉到0即完全过滤小道消息。
5. **过往世界杯数据因子** — 各队历史世界杯战绩指数(0~100:冠军、决赛、四强、参赛史),折算Elo修正(权重可调)。
6. **裁判层(`data/signals.json`)** — FIFA已于2026-04-09公布52人裁判池;**每场执法安排赛前1-2天才公布**,由每日更新逐场录入。裁判数据(场均黄牌/点球率/执法爆冷率)为中性统计,只小幅增加该场不确定性(最多±6%向五五开收敛)。**刻意不做"操盘嫌疑分"**:单个裁判每届只执法3-5场,样本无统计功效,指名暗示腐败属于诽谤。
7. **市场信号(资金流向)** — 有公开报道的大额资金/盘口异动时录入该场的市场隐含胜率,模型按可调权重(默认0.3)向市场混合;模型与市场背离>12%时打⚠警报。诚实说明:这检测的是"市场知道而模型不知道的信息"(阵容、动机、内幕,极少数情况是操纵),公开数据无法证明假球——真正的假球侦测(如Sportradar)依赖私有投注流数据。
8. **球员层(`data/players.json`)** — 每队核心球员(重要度q 1~10):
   - **伤病/停赛**:缺阵按重要度扣Elo(q10≈-45);"存疑"扣一半。
   - **红黄牌**:红牌或累计2黄→ban=N,只影响该队**接下来N场**(Match Predictor逐场生效,过后自动恢复);黄牌数追踪,FIFA规则1/4决赛后清零。
   - **动态状态值**:用赛事中实际数据(传球成功率、过人成功率、进球、助攻、出场时间)对比位置基准线算出状态分(-2~+2),状态好坏直接增减球队实力,并反哺更衣室士气。
   - **联赛强度**:每名球员标注所属俱乐部联赛(26个联赛系数表,英超=1.00、西甲0.96、意甲0.94、德甲0.93、法甲0.90、沙特联0.78、美职联0.77、卡塔尔联0.68等),有效实力 q_eff = q × 联赛系数(权重可调)。同为q8,英超的B.费有效8.0,沙特联的C罗有效6.2。
   - **年龄曲线**:每名球员有真实年龄,按位置设巅峰区间(门将27-34、后卫25-31、中场24-30、前锋23-29),巅峰外按距离衰减(41岁C罗×0.76,18岁亚马尔×0.93),权重可调。
   - **易受伤程度**:每人有伤病倾向评级0-3(铁人0如莫德里奇/范戴克,易伤2-3如德布劳内、阿拉巴),32岁以上额外+0.5;按"预期缺阵成本"折算Elo扣减。
   - **位置组合**:按位置组追踪可用实力占比,同一位置集中伤停产生**超线性**惩罚(英格兰双前锋同伤-121 Elo,远超前锋+中场各伤一人的-76)。整个位置组报销时球队卡会显示"⚠ positional imbalance"。
   - **世界杯经验**:每人记录参加过的世界杯届数(梅西/C罗×5、莫德里奇×4),封顶3届计分,单人最多+4 Elo——刻意做小,避免与球队底蕴因子双重计分。
   - **大赛心理(mental)**:默认全员0;只对有明确记录者设小值(E.马丁内斯+2点球大战专家、姆巴佩+2决赛帽子戏法、卢卡库-1、萨卡-0.5)。"大赛软脚"叙事多为噪声,故幅度极小且JSON可改。
   - **球星依赖度**:头号球员占全队有效实力>32%开始扣分(封顶-8)。注意:没有做"球星被盯防直接扣分"——盯防效应已含在Elo里,再扣是双重计分。
   - 每日自动更新会从比赛报告抓取这些数据(含转会后联赛修正);你也可以在Players页随时手动改状态/停赛场数。

比分用泊松进球模型;锦标赛按官方赛制(12组、8个最佳第三、FIFA 32强槽位约束、加时/点球)**在浏览器里**蒙特卡洛模拟,点"Re-run simulation"即可重算。你的滑块设置会保存在浏览器里。

## 如何更新数据

**方式A(在Cowork里,推荐)**:直接对我说"更新世界杯预测",我会抓最新Elo评分和比赛结果并重新生成仪表盘。已设置每日自动更新任务。

**方式B(在你自己电脑上,一键脚本)**:
```bash
python fetch_data.py --simulate
```
自动从 eloratings.net 拉最新评分、从ESPN公开API拉最新比分,然后重跑模拟并重新生成 index.html。无需任何API密钥。

**方式C(手动录入比分)**:编辑 `data/matches.json`,按现有格式添加一行(`"applied": false`),然后:
```bash
python engine/simulate.py --sims 20000
```

## 文件结构

```
worldcup2026/
├── index.html              # 预测仪表盘(打开这个)
├── dashboard_template.html # 仪表盘模板
├── fetch_data.py           # 一键数据更新脚本(本机运行)
├── engine/
│   ├── model.py            # Elo + ML校准 + 泊松比分模型
│   └── simulate.py         # 蒙特卡洛锦标赛模拟器
└── data/
    ├── teams.json          # 48队、分组、Elo评分快照
    ├── matches.json        # 已赛比分(动态追加)
    └── predictions.json    # 最新预测输出
```

## 说明

- 纯标准库Python,无需安装任何依赖。
- 第三名出线后的32强对阵采用官方槽位约束自动匹配(与FIFA附录C规则一致的合法分配)。
- 预测是概率估计,不构成任何投注建议。
