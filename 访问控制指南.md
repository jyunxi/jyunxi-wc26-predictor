# 登录门禁完整教程(Cloudflare Pages + Access)

目标效果:打开你的域名 → 先弹登录页 → 只有**你批准过的邮箱**能收到验证码进入 → 谁登录过你全看得到。免费支持50个用户,全程不写代码。

> 前提:已按《部署指南.md》把项目传到了GitHub仓库。本教程用Cloudflare托管(替代GitHub Pages,门禁必须配合Cloudflare托管才好用)。

---

## 第一步:注册Cloudflare并接入域名(10分钟)

1. 打开 dash.cloudflare.com,注册账号(邮箱+密码)。
2. 买域名,二选一:
   - **直接在Cloudflare买**(推荐):左侧菜单 **Domain Registration → Register Domains**,搜索想要的域名,成本价付款(.com约$10/年),买完自动接入,什么都不用配。
   - **已在别处买了**(阿里云/Namecheap等):Cloudflare首页点 **Add a site** 输入你的域名 → 选Free计划 → 它给你两个NS服务器地址 → 回到域名商后台把"DNS服务器/Nameservers"改成这两个 → 等生效(几分钟到几小时)。

## 第二步:用Cloudflare Pages托管网站(5分钟)

1. 左侧菜单 **Workers & Pages → Create → Pages → Connect to Git**。
2. 授权访问你的GitHub,选中 `wc26-predictor` 仓库。
3. 构建设置:Framework preset 选 **None**,Build command **留空**,Build output directory 填 `/`。点 **Save and Deploy**。
4. 一分钟后网站上线,地址形如 `wc26-predictor.pages.dev`。以后GitHub仓库每次更新(包括每日Actions自动更新),Pages都会自动重新发布,**不需要任何额外操作**。

## 第三步:绑定你的域名(3分钟)

1. 在这个Pages项目里点 **Custom domains → Set up a custom domain**。
2. 填 `wc.你的域名.com`(或者直接 `你的域名.com`),点确认——因为域名就在Cloudflare,DNS记录自动添加,HTTPS证书自动签发。
3. 几分钟后用域名访问,能打开就成功了。

## 第四步:挂上登录门禁(核心,10分钟)

1. 左侧菜单 **Zero Trust**(第一次进会让你取一个团队名,随便填,选Free计划)。
2. 进入 **Access → Applications → Add an application → Self-hosted**。
3. 配置:
   - Application name:`世界杯预测平台`
   - Session Duration:`24 hours`(登录一次管一天,可改7天)
   - **Public hostname**:填你第三步绑定的域名(如 `wc.你的域名.com`)
4. 下一步配置 **Policy(许可名单)**:
   - Policy name:`好友白名单`,Action:**Allow**
   - Include → Selector 选 **Emails** → 把允许访问的邮箱一个个填进去(先填你自己的)
   - 保存
5. 登录方式默认就是 **One-time PIN**(邮箱验证码),不用动。完成创建。

**生效了**:现在任何人打开你的域名都会先看到Cloudflare登录页;输入邮箱→在名单里的人收到6位验证码→输入即进;不在名单里的人**连验证码都收不到**。

## 第五步:日常管理(你的"审批"工作流)

- **批准新用户**:朋友把邮箱发给你 → Zero Trust → Access → Applications → 你的应用 → Policies → 编辑 → Emails里加一行 → 保存。10秒生效,这就是"创建需要你的授权"。
- **踢人**:同一个地方把邮箱删掉,对方下次就进不来了。
- **看谁在用**:Zero Trust → **Logs → Access**,每个人的邮箱、登录时间、IP、设备一目了然,这就是"注册/使用你能看见"。
- **人多了懒得一个个加**:Access → Access Groups 建一个组,Policy引用组,以后只维护组即可。

## 常见问题

- **每日自动更新会被门禁挡住吗?** 不会。GitHub Actions直接推代码到仓库,Pages自动重新部署,整个链路不经过登录页。
- **`pages.dev` 那个原始地址会绕过门禁吗?** 会!记得在Access应用里再加一条Public hostname把 `*.pages.dev` 域名也保护起来,或者在Pages设置里关闭 pages.dev 访问。
- **验证码邮件进垃圾箱?** 让朋友把 `noreply@notify.cloudflare.com` 加白。
- **能换成微信/Google登录吗?** 可以,Zero Trust → Settings → Authentication 里可加Google、GitHub等登录方式,白名单逻辑不变。
- **超过50人?** Cloudflare免费版上限50用户,够用;再多就要付费版或换自建方案(到时候找我做Workers+D1的注册审批系统)。

## 流程总结

```
朋友:发邮箱给你 ──→ 你:后台白名单加邮箱(10秒)
朋友:打开域名 → 输邮箱 → 收验证码 → 进入平台
你:Logs页面随时查看所有登录记录
```
