# 把预测平台发布成网站(GitHub Pages + 自定义域名)

总成本:**域名约 ¥60-80/年,其余全部免费**。发布后云端每天自动更新,你的电脑不用开机。

## 第一步:创建 GitHub 仓库(免费,5分钟)

1. 注册/登录 github.com,点 New repository,名字随意(如 `wc26-predictor`),选 **Public**,创建。
2. 把 `worldcup2026` 整个文件夹的内容上传到仓库(网页端可直接拖拽上传;或本地用git):
   ```bash
   cd worldcup2026
   git init && git add -A && git commit -m "init"
   git branch -M main
   git remote add origin https://github.com/你的用户名/wc26-predictor.git
   git push -u origin main
   ```
   > `.gitignore` 已配置好,`data/secrets.json`(API密钥)不会被上传。

## 第二步:配置API密钥(2分钟)

仓库页面 → Settings → Secrets and variables → Actions → New repository secret:
- Name: `ODDS_API_KEY`
- Secret: 你的The Odds API密钥

## 第三步:开启 GitHub Pages(1分钟)

仓库 → Settings → Pages → Source 选 **Deploy from a branch** → Branch 选 `main`、目录 `/ (root)` → Save。
一分钟后你的网站就在 `https://你的用户名.github.io/wc26-predictor/` 上线了。

## 第四步:每日自动更新(已配置好,无需操作)

`.github/workflows/daily-update.yml` 已就位:每天北京时间18:00,GitHub的服务器自动拉取最新Elo评分、ESPN比分、Pinnacle盘口(含24小时异动侦测),重跑2万次模拟,重新生成页面并发布。也可在仓库的 Actions 页面随时手动点 Run workflow。

> 注意:Actions自动更新覆盖 Elo/比分/盘口 这三类机械数据。球员逐场数据、裁判安排、更衣室新闻这三类需要理解新闻的更新,仍由Cowork里的每日任务维护——在Cowork更新后把改动推送到仓库即可同步上网(`git add -A && git commit -m sync && git push`)。

## 第五步:买域名并绑定(10分钟,唯一花钱的步骤)

**买哪家**:Cloudflare Registrar(成本价,约$10/年,推荐)、Namecheap、或国内阿里云/腾讯云(国内访问稳,但需实名;`.com`约¥70/年)。

**绑定**:
1. 在域名的DNS设置里加一条 **CNAME** 记录:主机名 `www`(或 `@` 用CNAME flattening),指向 `你的用户名.github.io`。
2. 回到仓库 Settings → Pages → Custom domain,填 `www.你的域名.com`,保存,勾选 **Enforce HTTPS**(证书自动签发,免费)。
3. 等几分钟DNS生效,你的预测平台就在自己的域名上了。

## 备选方案

- **Cloudflare Pages / Netlify**:同样免费,连GitHub仓库后自动部署,国内访问通常比github.io快;域名托管在Cloudflare的话一键绑定。
- **最懒方案**:不买域名,直接用 `github.io` 地址,零成本。

## 一句话提醒

页面含赔率与概率信息,仅供个人研究娱乐;如果要公开分享,别加任何引导投注的内容,各地对博彩信息的法规差异很大。
